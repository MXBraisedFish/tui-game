1. Lua SDK

5. 热重载
6. 调试覆盖层

8. API 兼容与迁移文档

最终定位可以概括为：

> **TUI Game 不需要成为功能缩水的通用游戏引擎，而应成为专门针对字符网格、终端交互和安全 Lua 模组的领域游戏引擎。**

你当前已经较好地完成了“终端怎么显示、怎么输入、宿主怎么运行”；下一阶段真正缺少的是“一个游戏在引擎里究竟是什么、拥有什么、如何被调度、如何安全地使用宿主能力”。这部分会成为 2.0 从宿主程序迈向游戏引擎的分界点。
