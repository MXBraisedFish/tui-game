# UI 框架层

## 当前阶段目标

从当前的 Page trait + 三个占位页面，升级为完整的 UI 框架：包含可复用的组件库、页面状态管理、页面间导航和数据流。UI 框架是玩家浏览游戏库、管理包、调整设置的主要交互界面。

---

## 服务项目清单

### 4-1 UiService 页面系统

- **职责**：管理页面注册、页面间导航、当前活动页面的渲染调度。这是 UI 框架的入口，运行时通过 UiService 与 UI 交互。
- **当前状态**：骨架存在。已有 `Page` trait（`key()` + `render()`）、`PageKey` 枚举（Home/Packages/Settings）、页面注册和导航方法。三个页面（HomePage、PackagesPage、SettingsPage）仅有占位文本。
- **待完善**：
  - 页面注册机制完善：支持页面注册时携带初始化参数。
  - 页面生命周期：`on_enter()`（页面展示时）、`on_leave()`（页面离开时）、`on_resize()`（终端尺寸变化时）回调。
  - 页面间数据传递：导航到新页面时可以携带上下文数据（如从列表页跳转到详情页时携带包 UID）。
  - 页面栈：支持前进/后退导航，而非仅左右切换。
  - 页面键扩展：未来的页面类型（包详情、游戏内覆盖 UI、确认对话框等）。

### 4-2 UIC 组件库

- **职责**：提供一组标准化的 UI 组件，每个组件封装自身的渲染逻辑和交互行为，页面通过组合组件构建界面。
- **当前状态**：不存在。当前占位页面直接在 `render()` 方法中调用 `draw_centered()` 写死文字。
- **待完善**：
  - **菜单（Menu）**：纵向列表，支持键盘上下选择、高亮当前项、确认/取消操作。
  - **可滚动列表（ScrollableList）**：当内容超出可视区域时支持滚动，显示滚动条指示器。
  - **对话框（ConfirmDialog）**：模态确认窗口，包含标题、正文、确认/取消按钮。
  - **开关列表（ToggleList）**：每项带有一个开关控件，用于设置页面（如启用/禁用某个包）。
  - **分屏面板（SplitPanel）**：左右或上下分屏，两个区域独立渲染。
  - **网格选择器（GridPicker）**：二维网格布局的选择器，用于游戏图标浏览等场景。
  - **文本输入（TextInput）**：单行文本输入组件，支持光标移动、编辑、粘贴。
  - **进度条（ProgressBar）**：显示加载进度或游戏内数值。
  - **标签页（TabBar）**：水平标签切换组件。

  每个组件应包含：
  - 自身的状态结构体
  - 接受输入事件并更新状态的方法
  - 渲染自身到给定 Canvas 区域的方法
  - 焦点管理：是否获得焦点、焦点切换顺序（Tab 键顺序）

### 4-3 UI 状态管理

- **职责**：管理每个页面的 UI 状态（如列表滚动位置、当前选中索引、折叠/展开状态等），确保页面在切换后返回时状态不丢失。
- **当前状态**：不存在。当前页面无任何状态存储。
- **待完善**：
  - 每个页面的状态结构体定义。
  - 状态持久化：页面离开时保留状态，返回时恢复。
  - 全局 UI 状态：当前焦点所在的页面和组件、全局模态状态（是否有对话框打开）。
  - 状态与渲染的分离：状态变更不直接触发渲染，由帧循环统一驱动渲染。

### 4-4 UI 页面实现

- **职责**：实现各具体页面，通过组合 UIC 组件构建完整的用户界面。
- **当前状态**：三个占位页面存在，无实际内容。
- **待完善**：

  **HomePage（首页）**：
  - 显示引擎 Logo / 标题
  - 显示已安装游戏数量、屏保数量
  - 快捷入口：游戏列表、设置
  - 显示最近玩过的游戏

  **GameListPage / PackagesPage（包列表页）**：
  - 从 PackageService 读取已安装的游戏包列表
  - 每个条目显示：图标、标题、作者、版本、启用/禁用状态
  - 支持排序（按标题、按最近使用、按安装时间）
  - 选择条目进入详情页/启动游戏
  - 支持搜索/过滤

  **ModScreensaverListPage（屏保/Boss 列表页）**：
  - 类似 GameListPage，展示屏保和 Boss 包

  **SettingsPage（设置页）**：
  - 分类设置：显示设置、语言设置、键位设置、存储管理、安全设置
  - 每个设置分类使用开关列表/选择器组件
  - 键位设置特殊处理：进入独立的键位编辑界面

  **PackageDetailPage（包详情页）**：
  - 显示包完整信息：描述、作者、版本、API 兼容性
  - 启用/禁用操作
  - 启动游戏 / 预览屏保按钮
  - 运行时需求显示（最小终端尺寸等）

## 旧架构参考

### UI 页面（`old_src/host_engine/runtime/ui/pages/`）
旧架构实现了以下页面，每个页面是一个独立模块：
- **home_page**：引擎首页，包含标题和导航入口。
- **game_list_page**：游戏列表，从 PackageRegistry 读取数据，支持排序和过滤。
- **mod_screensaver_list_page**：屏保和 Boss 列表。
- **settings_page**：设置页面，包含多个子页面（display、language、memory、security、keybinds）。
- **warnings_page**：引擎警告信息汇总页面。
- **storage_details_page**：存储使用详情页面。

### UI 组件（`old_src/host_engine/runtime/ui/components/`）
旧架构有 7 个可复用组件：
- **confirm_dialog**：确认对话框，模态覆盖在当前页面之上。
- **grid_picker**：二维网格选择器，用于游戏图标浏览。
- **menu**：纵向菜单列表。
- **scrollable_list**：可滚动列表，支持键盘和鼠标滚动。
- **split_panel**：分屏面板。
- **toggle_list**：带开关的选项列表。
- 另有一个通用的组件 trait 或基类。

### UI 状态（旧架构相关模块）
旧架构使用了约 17 个状态结构体管理各页面和组件的状态。这些状态在页面切换时保留，页面返回时恢复。

旧架构的 UI 系统直接访问 PackageRegistry、ProfileStore 等服务做数据读取，这在当时是合理的，但新架构应改为通过请求方式获取数据（UiService 请求数据，而非直接读取内部状态）。

## 完成后可验证的可用项

1. 首页正确显示引擎名称、已安装包数量和快捷入口。
2. 游戏列表页从 PackageService 读取并展示所有已安装游戏。
3. 设置页可以浏览所有设置分类，修改开关项后保存生效。
4. 对话框弹出时阻止背景页面的按键响应（模态行为）。
5. 列表页内容超出屏幕时可以滚动，滚动指示器显示当前位置。
6. 页面切换后返回，之前的滚动位置和选中项保持不变。
7. 终端 resize 后页面重新布局，不出现错位。
